Message-Digest Algorithm MD5(MD5)

加密方法

补位与拓展

将数据按位补充至 bits_length $$\equiv$$ 448 (mod 512)

补位方法为在信息后补一个 1 和 i 个 0 （$$0\leqslant i \leqslant 511$$）

其后接上bits_length mod $2^{64}$ 得到的二进制数

初始化MD缓存器

MD5运算要用到一个128位的MD5缓存器，用来保存中间变量和最终结果。该缓存器又可看成是4个32位的寄存器A、B、C、D，初始化为 :（小端序）

A ： 01 23 45 67 (0x67452301)
B： 89 AB CD EF (0xEFCDAB89)
C： FE DC BA 98 (0x98BADCFE)
D： 76 54 32 10 （0x10325487)

处理数据段

四个函数

四个操作
$$
FF(a,b,c,d,m_i,s,tj) 表示a=b+((a+(F(b,c,d)+m_i+\lambda_j )<<< s)\
GG(a,b,c,d,m_i,s,tj) 表示 a=b+((a+(G(b,c,d)+m_i+\lambda_j)<<< s) \
HH(a,b,c,d,m_i,s,tj) 表示 a=b+((a+(H(b,c,d)+m_i+\lambda_j)<<< s) \
II(a,b,c,d,m_i,s,tj) 表示a=b+((a+(I(b,c,d)+m_i+\lambda_j)<<< s) \其中，
m_i为消息第i个子分组，i=0,1,…,15\
<<< s表示循环左移s位\
第j步中，\lambda_j=2^{32}*|[\sin j]|\

x循环左移s位:\
( s << x ) | ( s >> (32 - x) )
$$

每轮使用FF，GG，HH，II中的一种操作；

一轮中，a,b,c,d的使用顺序轮转;

消息分以512位为一分组进行处理，对每个512位分组，进行

每一个分组进行上述4轮共64次计算后，将A，B，C，D分别加上计算得到的a，b，c，d。当做新的A，B，C，D，并将这4个变量赋值给a,b,c,d再进行下一分组的运算。由于填充后的消息长度为(N+1)*512，则共需计算N+1个分组。计算所有数据分组后，这4个变量为最后的结果，即MD5值。
